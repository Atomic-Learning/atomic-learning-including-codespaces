In general, all that is needed for a Codespace is a GitHub repository. As every content page in the Atomic Learning platform is created from a Codespace, the possibility of creating a Codespace for each content page already exists by default.

However, you can make the Codespace more obvious and accessible to users by adding the codespace tag to the metadata file, as in the metadata file for this page. This will add a banner to the top of the content page, such as the one on this page, which makes it easier for readers to get to the Codespace.

If this is all that is done, the Codespace will be simple in the sense that there will not be any programs or VS Code extensions pre-installed when the user opens it. Instead, it will be a basic Linux environment, which may be sufficient for simple examples and exercises. However, in many cases, you will want to include a Dockerfile or similar to define and customise the environment the reader will experience when they open the Codespace.

Typically, when adding a Codespace to a content page, the page will contain a description of the purpose of the Codespace and how the reader should interact with it.
